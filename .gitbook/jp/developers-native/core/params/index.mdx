# `x/params`

> 注: Paramsモジュールは非推奨となり、各モジュールが自身のパラメータを保持する方式が推奨されています。

## 概要

paramsパッケージは、グローバルに利用可能なパラメータストアを提供します。

主要な型はKeeperとSubspaceの2つです。Subspaceはパラメータストアのための隔離された名前空間であり、キーはあらかじめ設定されたスペース名で接頭辞付けされます。Keeperはすべての既存スペースにアクセスする権限を持ちます。

Subspaceは、他のKeeperが変更できないプライベートなパラメータストアを必要とする個々のKeeperによって使用できます。params Keeperは、プロポーザルが可決された場合に任意のパラメータを変更するために、`x/gov`ルーターへのルートを追加するのに使用できます。

以下では、masterおよびユーザーモジュールに対するparamsモジュールの使い方を説明します。

## Keeper

アプリケーションの初期化段階で、`Keeper.Subspace`を使用して他のモジュールのKeeperに[subspaces](#subspace)を割り当てることができ、`Keeper.spaces`に格納されます。これらのモジュールは、`Keeper.GetSubspace`を通じて固有のパラメータストアへの参照を持てます。

例:

```go
type ExampleKeeper struct {
	paramSpace paramtypes.Subspace
}

func (k ExampleKeeper) SetParams(ctx sdk.Context, params types.Params) {
	k.paramSpace.SetParamSet(ctx, &params)
}
```

## Subspace

`Subspace`は、パラメータストアの接頭辞付きサブスペースです。パラメータストアを使用する各モジュールは、アクセス権限を隔離するために`Subspace`を取得します。

### Key

パラメータキーは、人間が読める英数字の文字列です。キー`"ExampleParameter"`に対するパラメータは、`[]byte("SubspaceName" + "/" + "ExampleParameter")`に格納されます（`"SubspaceName"`はサブスペースの名前）。

サブキーは、主キーと組み合わせて使用される二次的なパラメータキーです。サブキーは、グループ化や、実行時の動的なパラメータキー生成に使用できます。

### KeyTable

使用するすべてのパラメータキーは、コンパイル時に登録される必要があります。`KeyTable`は本質的に`map[string]attribute`であり、`string`はパラメータキーです。

現在、`attribute`は、提供されたキーと値が互換性を持ち登録されていることを確認するためのパラメータ型を示す`reflect.Type`と、値を検証する関数`ValueValidatorFn`で構成されています。

`KeyTable`に登録する必要があるのは主キーのみです。サブキーは主キーのattributeを継承します。

### ParamSet

モジュールはしばしばパラメータをprotoメッセージとして定義します。生成された構造体は`ParamSet`インターフェースを実装することで、次のメソッドで使用できます:

* `KeyTable.RegisterParamSet()`: 構造体内のすべてのパラメータを登録します
* `Subspace.{Get, Set}ParamSet()`: 構造体に対するGetとSetを行います

`GetParamSet()`を使用するために、実装はポインタである必要があります。
