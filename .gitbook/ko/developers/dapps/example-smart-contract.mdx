---

이 짧은 시리즈에서는 Injective에서 dApp을 구축하는 것이 얼마나 쉬운지 보여드리겠습니다. 오픈 소스 [dApp](https://github.com/InjectiveLabs/injective-simple-sc-counter-ui)이 있으며 모든 사람이 참조하고 Injective에서 구축하는 데 사용할 수 있습니다. Next, Nuxt 및 Vanilla Js용 예제가 있습니다. 처음부터 시작하려는 사람에게 이것이 적합한 곳입니다.

이 예제에서는 injective-ts 모듈을 사용하여 Injective Chain에 배포된 예제 스마트 컨트랙트와 연결하고 상호작용하는 것을 구현합니다.

시리즈에는 다음이 포함됩니다:

- API 클라이언트 및 환경 설정,
- Chain 및 Indexer API에 연결,
- 사용자 지갑에 연결하고 주소 가져오기,
- 스마트 컨트랙트 조회(이 경우 스마트 컨트랙트의 현재 count 가져오기),
- 컨트랙트 상태 수정(이 경우 count를 1 증가시키거나 특정 값으로 설정),

## 설정

먼저 원하는 UI 프레임워크를 구성하세요. 구성에 대한 자세한 내용은 여기에서 찾을 수 있습니다.

dex를 시작하려면 API 클라이언트와 환경을 설정해야 합니다. DEX를 구축하기 위해 Injective Chain과 Indexer API 모두에서 데이터를 조회합니다. 이 예제에서는 기존 **Testnet** 환경을 사용합니다.

먼저 데이터를 조회하는 데 필요한 클래스를 설정합니다.

스마트 컨트랙트와 상호작용하기 위해 `@injectivelabs/sdk-ts`의 `ChainGrpcWasmApi`를 사용합니다. 또한 `@injectivelabs/networks`에서 사용할 Network Endpoint(Mainnet 또는 Testnet)가 필요합니다.

예제:

```js
//filename: services.ts
import { Network, getNetworkEndpoints } from "@injectivelabs/networks";
import { ChainGrpcWasmApi } from "@injectivelabs/sdk-ts/client/wasm";

export const NETWORK = Network.Testnet;
export const ENDPOINTS = getNetworkEndpoints(NETWORK);

export const chainGrpcWasmApi = new ChainGrpcWasmApi(ENDPOINTS.grpc);
```

그런 다음 사용자가 DEX에 연결하고 트랜잭션에 서명할 수 있도록 지갑 연결을 설정해야 합니다. 이를 위해 사용자가 다양한 지갑 제공자와 연결하고 Injective에서 트랜잭션에 서명할 수 있게 해주는 `@injectivelabs/wallet-strategy` 패키지를 사용합니다.

`@injectivelabs/wallet-strategy`의 주요 목적은 개발자에게 Injective에서 다양한 지갑 구현을 가질 수 있는 방법을 제공하는 것입니다. 이러한 모든 지갑 구현은 동일한 `ConcreteStrategy` 인터페이스를 노출하므로 사용자는 특정 지갑에 대한 기본 구현을 알 필요 없이 이러한 메서드를 사용할 수 있습니다.

시작하려면 다양한 지갑을 바로 사용할 수 있는 WalletStrategy 클래스의 인스턴스를 만들어야 합니다. walletStrategy 인스턴스에서 `setWallet` 메서드를 사용하여 사용되는 현재 지갑을 전환할 수 있습니다(참고: `setWallet`은 async이며 `await`가 필요합니다). 기본값은 `Metamask`입니다.

```ts
// filename: wallet.ts
import { ChainId, EvmChainId } from "@injectivelabs/ts-types";
import { WalletStrategy } from "@injectivelabs/wallet-strategy";

const chainId = ChainId.Testnet; // Injective Testnet Chain ID
const evmChainId = EvmChainId.TestnetEvm; // Injective Evm Testnet Chain ID

export const evmRpcEndpoint = `https://eth-sepolia.g.alchemy.com/v2/${process.env.APP_EVM_RPC_KEY}`;

export const walletStrategy = new WalletStrategy({
  chainId,
  evmOptions: {
    evmChainId,
    rpcUrl: evmRpcEndpoint,
  },
});
```

Ethereum 네이티브 지갑을 사용하지 않으려면 `WalletStrategy` 생성자 내에서 `evmOptions`를 생략하면 됩니다.

마지막으로 Injective에서 전체 트랜잭션 흐름(준비 + 서명 + 브로드캠스트)을 수행하기 위해 MsgBroadcaster 클래스를 사용합니다.

```js
import { Network } from "@injectivelabs/networks";
export const NETWORK = Network.Testnet;

export const msgBroadcastClient = new MsgBroadcaster({
  walletStrategy,
  network: NETWORK,
});
```

## 사용자 지갑에 연결

사용자 지갑과의 연결을 처리하기 위해 `WalletStrategy`를 사용하므로 사용자 주소 가져오기, 트랜잭션 서명/브로드캠스트 등과 같은 사용 사례를 처리하기 위해 해당 메서드를 사용할 수 있습니다. wallet strategy에 대해 더 자세히 알아보려면 문서 인터페이스와 `WalletStrategy`가 제공하는 메서드를 탐색할 수 있습니다.

참고: `setWallet` 메서드를 사용하여 `WalletStrategy` 내에서 "active" 지갑을 전환할 수 있습니다(async이며 `await`가 필요합니다).

```ts
// filename: WalletConnection.ts
import {
  WalletException,
  UnspecifiedErrorCode,
  ErrorType,
} from "@injectivelabs/exceptions";
import { Wallet } from "@injectivelabs/wallet-base";
import { walletStrategy } from "./Wallet.ts";

export const getAddresses = async (wallet: Wallet): Promise<string[]> => {
  await walletStrategy.setWallet(wallet);

  const addresses = await walletStrategy.getAddresses();

  if (addresses.length === 0) {
    throw new WalletException(
      new Error("이 지갑에 연결된 주소가 없습니다."),
      {
        code: UnspecifiedErrorCode,
        type: ErrorType.WalletError,
      }
    );
  }

  if (!addresses.every((address) => !!address)) {
    throw new WalletException(
      new Error("이 지갑에 연결된 주소가 없습니다."),
      {
        code: UnspecifiedErrorCode,
        type: ErrorType.WalletError,
      }
    );
  }

  // Ethereum 네이티브 지갑을 사용하는 경우 'addresses'는 hex 주소입니다
  // Cosmos 네이티브 지갑을 사용하는 경우 'addresses'는 bech32 injective 주소입니다,
  return addresses;
};
```

## 조회

초기 설정이 완료되면 이전에 만든 chainGrpcWasmApi 서비스를 사용하고 스마트 컨트랙트에서 get_count를 호출하여 현재 count를 가져오기 위해 스마트 컨트랙트를 조회하는 방법을 살펴보겠습니다.

```ts
function getCount() {
  const response = (await chainGrpcWasmApi.fetchSmartContractState(
    COUNTER_CONTRACT_ADDRESS, // 컨트랙트 주소
    toBase64({ get_count: {} }) // 조회를 Base64로 변환해야 합니다
  )) as { data: string };

  const { count } = fromBase64(response.data) as { count: number }; // 응답을 Base64에서 변환해야 합니다

  return count; // 현재 counter 값을 반환합니다.
}
```

이러한 함수(`getCount` 또는 우리가 만드는 다른 함수)가 있으면 애플리케이션 어디에서나 호출할 수 있습니다(일반적으로 Nuxt의 Pinia, React의 Context providers 등과 같은 중앙 집중식 상태 관리 서비스).

## 상태 수정

다음으로 `count` 상태를 수정합니다. 이전에 만든 `Broadcast Client`와 `@injectivelabs/sdk-ts`의 `MsgExecuteContractCompat`를 사용하여 체인에 메시지를 보내 수행할 수 있습니다.

이 예제에서 사용하는 스마트 컨트랙트는 상태를 변경하기 위한 2가지 메서드가 있습니다:

- `increment`
- `reset`

`increment`는 count를 1 증가시키고, `reset`은 count를 주어진 값으로 설정합니다. `reset`은 스마트 컨트랙트의 생성자인 경우에만 호출할 수 있습니다.

이러한 함수를 호출하면 메시지/트랜잭션에 서명하고 브로드캠스트하기 위해 지갑이 열립니다.

먼저 count를 증가시키는 방법을 보겠습니다.

```js
// 메시지 준비

const msg = MsgExecuteContractCompat.fromJSON({
  contractAddress: COUNTER_CONTRACT_ADDRESS,
  sender: injectiveAddress,
  msg: {
    increment: {}, // 메서드에 파라미터가 없으면 빈 객체를 전달합니다
  },
});

// 메시지 서명 및 브로드캠스트

const response = await msgBroadcastClient.broadcast({
  msgs: msg, // 여기에 배열을 사용하여 여러 메시지를 전달할 수 있습니다. 예: [msg1,msg2]
  injectiveAddress: injectiveAddress,
});

console.log(response);
```

이제 counter를 특정 값으로 설정하는 방법 예제를 보겠습니다. 이 스마트 컨트랙트에서는 스마트 컨트랙트의 생성자만 count를 특정 값으로 설정할 수 있습니다.

```js
// 메시지 준비

const msg = MsgExecuteContractCompat.fromJSON({
  contractAddress: COUNTER_CONTRACT_ADDRESS,
  sender: injectiveAddress,
  msg: {
    reset: {
      count: parseInt(number, 10), // number 변수를 여기서 파싱합니다. 일반적으로 input에서 오며 항상 문자열을 제공하는데, 대신 숫자를 전달해야 하기 때문입니다.
    },
  },
});

// 메시지 서명 및 브로드캠스트

const response = await msgBroadcastClient.broadcast({
  msgs: msg,
  injectiveAddress: injectiveAddress,
});

console.log(response);
```

### 전체 예제

Vanilla JS의 전체 예제를 보겠습니다(Nuxt 및 Next와 같은 특정 프레임워크의 예제는 [여기](https://github.com/InjectiveLabs/injective-simple-sc-counter-ui)에서 찾을 수 있습니다)

```js
import { Web3Exception } from "@injectivelabs/exceptions"
import { WalletStrategy } from "@injectivelabs/wallet-strategy"
import { ChainGrpcWasmApi } from "@injectivelabs/sdk-ts/client/wasm"
import { Network, getNetworkEndpoints } from "@injectivelabs/networks"
import { getInjectiveAddress } from "@injectivelabs/sdk-ts/utils"

const chainId = ChainId.Testnet // Injective Testnet Chain ID
const evmChainId = EvmChainId.TestnetEvm // Injective Evm Testnet Chain ID

export const evmRpcEndpoint = `https://eth-sepolia.g.alchemy.com/v2/${process.env.APP_EVM_RPC_KEY}`

const NETWORK = Network.Testnet
const ENDPOINTS = getNetworkEndpoints(NETWORK)

const chainGrpcWasmApi = new ChainGrpcWasmApi(ENDPOINTS.grpc)

export const walletStrategy = new WalletStrategy({
  chainId,
  evmOptions: {
    evmChainId,
    rpcUrl: evmRpcEndpoint,
  },
})

export const getAddresses = async (): Promise<string[]> => {
  const addresses = await walletStrategy.getAddresses()

  if (addresses.length === 0) {
    throw new Web3Exception(
      new Error("이 지갑에 연결된 주소가 없습니다.")
    )
  }

  return addresses
}

const msgBroadcastClient = new MsgBroadcaster({
  walletStrategy,
  network: NETWORK,
})

const [address] = await getAddresses()
const injectiveAddress = getInjectiveAddress(getInjectiveAddress)

async function fetchCount() {
  const response = (await chainGrpcWasmApi.fetchSmartContractState(
    COUNTER_CONTRACT_ADDRESS, // 컨트랙트 주소
      toBase64({ get_count: {} }) // 조회를 Base64로 변환해야 합니다
    )) as { data: string }

  const { count } = fromBase64(response.data) as { count: number } // 응답을 Base64에서 변환해야 합니다

  console.log(count)
}

async function increment(){
    const msg = MsgExecuteContractCompat.fromJSON({
    contractAddress: COUNTER_CONTRACT_ADDRESS,
    sender: injectiveAddress,
    msg: {
        increment: {},
        },
    })

    // 메시지 서명 및 브로드캠스트

    await msgBroadcastClient.broadcast({
        msgs: msg,
        injectiveAddress: injectiveAddress,
    })
}

async function main() {
    await fetchCount() // 출력: {count: 5}
    await increment() // 트랜잭션에 서명하고 브로드캠스트하기 위해 지갑이 열립니다
    await fetchCount() // count가 이제 6입니다. 출력: {count: 6}
}

main()

```
