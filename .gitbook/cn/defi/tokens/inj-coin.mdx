title: INJ 代币
---

INJ 是驱动 Injective 及其更广泛生态系统的原生资产。INJ 的每个组件都经过精心设计，以培育一个繁荣的 Web3 生态系统。作为区块链的原生资产，INJ 在促进 Injective 上的各种操作中发挥着核心作用。INJ 是 Injective 自定义实现的 Tendermint 权益证明（PoS）共识框架的组成部分，对于通过质押保护网络至关重要。此外，INJ 作为 Injective 的治理 token，并在更广泛的 Injective 生态系统中作为交换媒介。值得注意的是，INJ 通过利用核心 Injective 模块，通过创新的销毁和动态供应机制来设计通缩特性，从而与其他 PoS 链上的原生资产区分开来。

### 基础面额

INJ 使用 [Atto](https://en.wikipedia.org/wiki/Atto-) 作为基础面额，以保持与 Ethereum 的一致性。

```
1 inj = 1×10⁻¹⁸ INJ
```

这与 Ethereum 的面额匹配：

```
1 wei = 1x10⁻¹⁸ ETH
```

### Injective 代币经济学和效用

#### 1. 安全性和质押

Injective 通过质押来保护，这是 INJ 的一个重要用例。验证者和委托者可以通过质押自由参与 Injective 网络。验证者在 Injective 上运行节点，委托者可以将 INJ 分配给他们选择的特定节点。质押的 INJ 通过惩罚和奖励系统确保安全，从而实现强大的去中心化环境。

如果验证者出现恶意行为或未能有效履行职责，其质押的 INJ 将被罚没。此外，INJ 用于奖励验证者参与交易验证和区块创建。验证者的奖励包括新铸造的 INJ（区块奖励）和相关交易费用的一部分。

INJ 持有者也可以参与质押，而不必运行节点来赚取验证者奖励的一部分。为此，用户将 INJ 委托给验证者，这可以通过支持的浏览器钱包或直接通过 Injective Hub 完成。作为锁定 INJ 的回报，用户按比例获得验证者 INJ 奖励的一部分，减去所选验证者收取的费用（佣金）。如果委托的验证者发生罚没事件，用户质押的 INJ 也会被罚没。这确保验证者和委托者都致力于网络的整体安全。

除了保护 Injective 链之外，INJ 还通过 Electro Chains 将其安全服务扩展到更广泛的生态系统。这些基于 Injective 的 rollup 提供了众多技术优势，例如支持多个虚拟机，如 inEVM 所示。由于这些 rollup 结算到 Injective，INJ 为这些网络提供了基础安全层。这种互联的安全框架强调了 INJ 在维护 Injective 网络以及 Electro Chains 多样化生态系统的完整性和稳健性方面的关键作用。

#### 2. 治理

INJ 用于链上所有参数的社区主导治理。Injective 独特地拥有智能合约上传的权限层，这意味着质押者社区必须投票才能在主网上实例化智能合约。这使社区能够直接治理 Injective 整体的所有参数。

对于治理，INJ 用于提案创建和对活跃提案的 token 加权投票。作为垃圾邮件威慑，Injective 要求以 INJ 支付的最低存款，以便提案进入投票阶段。这个存款门槛可以完全由提案者满足，也可以由其他用户向提案存款累积满足。如果在最大存款期限到期时未达到最低存款金额，提案将自动被拒绝，存款将被销毁。此外，如果提案在投票期限到期时未通过，提案存款将被销毁。

提案投票在预设的投票期间进行，该期间通过治理设置并统一适用于所有治理投票。在投票过程中，只有质押的 INJ 有资格参与投票。因此，只有验证者和委托者可以对活跃提案投票。投票权是 token 加权的，意味着 1 INJ 等于 1 票。委托者不需要积极参与治理来维持其状态。但是，他们可以选择直接对提案投票。如果委托者不投票，他们的投票权将自动由他们委托的验证者继承，用于该特定投票事件。

INJ 用于治理链的所有方面，包括：

* Auction 模块参数
* Exchange 模块自定义提案和参数
* Insurance 模块参数
* Oracle 模块自定义提案
* Peggy 模块参数
* Wasmx 模块参数
* 软件升级
* Cosmos-SDK 模块参数，包括 [auth](https://docs.cosmos.network/main/modules/auth#parameters)、[bank](https://docs.cosmos.network/main/modules/bank)、[crisis](https://docs.cosmos.network/main/modules/crisis)、[distribution](https://docs.cosmos.network/main/modules/distribution)、[gov](https://docs.cosmos.network/main/modules/gov)、[mint](https://docs.cosmos.network/main/modules/mint)、[slashing](https://docs.cosmos.network/main/modules/slashing) 和 [staking](https://docs.cosmos.network/main/modules/staking) 模块。

有关治理流程的完整详情，请参阅[这里](https://blog.injectiveprotocol.com/injective-governance-proposal-procedure)。

#### 3. 交换媒介

INJ 用作默认资产，以促进区块链上各方之间商品和服务的买卖。常见的例子包括支付交易费用（gas）、买卖 NFT、支付交易费用或将资产作为抵押品存入。虽然大多数商品和服务可以用任何资产计价，但 Injective 上产生的所有交易费用都以 INJ 支付。此外，通过 exchange 模块利用 Injective 共享流动性层的应用产生的所有协议收入都以 INJ 累积。

#### 4. Exchange dApp 激励

exchange 协议实施了 0.1% 的 maker 和 0.2% 的 taker 全局最低交易费用。作为鼓励 exchange dApp 在 exchange 协议上获取交易活动的激励机制，将订单发起到共享订单簿的 exchange dApp 将获得其获取的所有订单产生的交易费用的 40%。

#### 5. Exchange 费用价值累积

剩余 60% 的 exchange 费用将经历链上回购销毁事件，其中汇总的 exchange 费用篮子将拍卖给出价最高者以换取 INJ。此拍卖的 INJ 收益随后被销毁，从而使 INJ 总供应量通缩。

有关拍卖机制的更多详情，请参阅[这里](../community-buyback/)。

#### 6. 衍生品的支持抵押品

INJ 可以作为稳定币的替代品，用作 Injective 衍生品市场的保证金和抵押品。在某些衍生品市场中，INJ 还可以用作保险池质押的支持抵押品，质押者可以在其锁定的 token 上赚取利息。
